Getting started with Toolkit v3.4.1

Overview

The Directgov is a resource for designers and developers involved in the delivery of services through the Directgov brand and not through the Directgov infrastructure.

This guidance is to be used alongside discussions with the Directgov Publishing team and is continually evolving, so please check regularly for updates. If you have any questions or points you wish to clarify please don't hesitate to get in touch.

Please familiarise yourself with the Toolkit in order to apply the Directgov brand to your application.

Before you start using the Toolkit

  1. Please contact us to discuss your requirements - Richard Watson on +44 (0)7973 655 754
  2. Please ensure that you only use the template examples provided and not these framework pages

What is in the Toolkit?

The Toolkit is made up of two components:

  1. General presentation guidelines outlined in this framework and
  2. template examples including HTML, CSS and Javascript code, that you would use as a starting point to your development.

Template examples

Templates examples are on on every page under Template examples in the right hand column - top right of each page.

Please note the template examples are intended as a starting point to your development with certain elements in the design and code fixed for consistency. If in doubt or if the examples don't fit your requirements, please contact us for advice.

What is the purpose of the Toolkit?

The Toolkit provides guidance and actual code for developers to use when creating their White Labeled Services (WLS). This ensures consistency across any tool or transaction including correct behaviour, alignment, spacing and style.

What are tools and transactions?

Tools and transactions are standalone applications, hosted and run outside of the Directgov core infrastructure.

Providing a consistent and seemless user experience is key to citizens increasingly interacting with Government online.

The white label approach creates significant challenges in delivering a consistent brand and customer experience throughout the citizen's online journey. This is particularly the case with existing applications that have to be repurposed to accept the Directgov brand.

This site provides guidance and suggestions to enable White Labeled Services to conform to the Directgov toolkit so that citizens understand and become increasingly comfortable transacting with Government online.

Toolkit versions

The toolkit has been released as the following versions.

Transactions Toolkit v1.0

Legacy Toolkit providing guidance and examples for original Directgov design.

Transactions Toolkit v2.0 BETA

A new look and feel was adopted by Directgov and new Toolkit templates published.

Transactions Toolkit v2.1 BETA

Improvements to Toolkit v2.0 were developed to ensure the Toolkit examples were easily scalable and easy to use.

Transactions Toolkit v3.0

New code developed to ensure the Toolkit examples were further scalable and adhered to Directgov's accessibility standards. This included thorough accessibility and technical audits by The Shaw Trust and Nomensa.

Transactions Toolkit v3.4

With feedback received from a number of projects the Toolkit templates were updated for further ease of use. This included new template examples and Javascript alternative to rounded corners.

Toolkit v3.4 release in December 2008.

Transactions Toolkit v3.4.1

Search, Simple data table and Dynamic data table results templates added. Addition of Close window examples.

Toolkit v3.4.1 release in July 2009.
