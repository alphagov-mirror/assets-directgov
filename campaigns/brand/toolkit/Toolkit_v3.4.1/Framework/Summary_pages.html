Summary/confirmation pages

Summary pages or confirm screens enable the user to view information they have inputted and either amend the details or verify them prior to completing the transaction or displaying the result set.

  • Information submitted by users is summarised and presented back, normally presented within a definition list, for correction or confirmation.
  • The user may go back and edit, or proceed with the transaction.

Providing links to edit or delete

  • Clicking on an edit link will take the user back to the original step where they inputted the information. All information they originally inputted should be displayed for them to edit.
  • Links to edit or change information inputted are usually provided on the page prior to the last step or milestone step within the transaction.
  • The edit or delete link must be associated via visual proximity, near the field to which it refers.
  • If allowing a user to delete a data element via the delete function, it is recommended that upon selecting to delete an 'Are you sure?' type page is provided which outlines any consequences the user should be informed of. The user will then have the option to continue or cancel the delete action.
