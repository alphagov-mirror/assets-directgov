Form fields

A form may contain a variety of data input fields for users. It is important to ensure that the form looks readable at resolutions widths ranging from 800 up to 1280 pixels.

Principles of form fields

  • Fieldsets will be used to group form elements that share a common function or heading. An example would be a group of radio buttons or three separate input fields for day/month/year. Do not encompass your entire form within a fieldset and do not nest fieldsets.
  • All forms will have a grey keyline border, with rounded corners and a white background.
  • When form elements have a heading title the form should be styled to have a grey heading bar containing the heading title. When the form elements do not have a heading title the form elements will be contained within the default grey keyline border.
  • All form inputs must be associated with a label. In the case of text, text area and select inputs the label will appear immediately above the input. For radio buttons and checkboxes the label will appear to the right of the input.
  • Radio buttons and checkboxes should be contained within a fieldset. A legend will appear directly above these inputs.
  • Compulsory fields must be denoted by an asterisk at the start of the field label.

Denoting a compulsory field

  • Compulsory fields are denoted by an asterisk.
  • Asterisks are placed at the start of the question.
  • Users must be notified at the top of the page what the asterisk means.
  • Asterisks most commonly used to show which name, address and other question fields are compulsory. Decision trees where all questions are compulsory do not usually use asterisks.
  • Be realistic about which fields you make mandatory for users to complete.

Examples

Text input

A text input is used when you need a small amount of text, for example, 'first name'.

Textarea

A textarea should be used when you need a larger amount of text, for instance a paragraph of information.

Radio buttons

Check boxes

Select menus

Like radio buttons, a select menu allows the user to choose one option from several provided, for example, "Country of birth".

Combinations of dependant select menus

It may be relevant at times to link select menus. So if the user selects option 2 (Renault) from menu 1 (make of car), then a filtered set of options (Renault models) populates menu 2 (model of car).

To achieve this it is possible to use javascript. However, it is vital that if the user has javascript turned off, they are still able to select relevant options within the dependant menus.
